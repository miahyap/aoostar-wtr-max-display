# aoostar-lcd — Unraid plugin for the AOOSTAR WTR MAX integrated display

Drives the WTR MAX (and GEM12+ PRO) case LCD on Unraid using the
open-source, reverse-engineered [aoostar-rs](https://github.com/zehnm/aoostar-rs)
driver — **no vendor software required**. Two small Rust binaries
(~17 MB total, running from RAM):

- **`aster-sysinfo`** — collects CPU / memory / network / temperature /
  disk sensors into a text file in tmpfs every few seconds
- **`asterctl`** — renders a sensor panel (AOOSTAR-X theme format) and
  pushes it to the LCD over its USB serial port (USB ID `0416:90a1`)

The plugin wires them into Unraid properly:

- Starts at boot (independent of the array — nothing touches `/mnt`)
- Settings page under *Settings → Utilities → AOOSTAR LCD*: status,
  start/stop, display on/off, sensor listing, and options (autostart,
  network interface, refresh intervals, smartctl on/off)
- Panel config and sensor mapping persist on the flash drive at
  `/boot/config/plugins/aoostar-lcd/cfg/`
- The asterctl release tarball is cached on flash, so reboots reinstall
  fully offline
- Disk-friendly defaults: smartctl temperature polling is **off**
  (it would wake spun-down disks) and disk usage refresh is 30 min

## Install

1. Copy the plugin (and optionally the verified driver tarball for a
   fully offline install) to the Unraid flash drive:

   ```sh
   scp aoostar-lcd.plg root@tower:/boot/config/plugins/
   # optional but recommended (skips the GitHub download):
   ssh root@tower mkdir -p /boot/config/plugins/aoostar-lcd/packages
   scp asterctl-v0.2.0-6-g2f4d959-Linux-x64.tar.gz \
       root@tower:/boot/config/plugins/aoostar-lcd/packages/
   ```

   (replace `tower` with your server's hostname/IP)

2. Install it — either in the web UI via **Plugins → Install Plugin**,
   entering `/boot/config/plugins/aoostar-lcd.plg`, or over SSH:

   ```sh
   plugin install /boot/config/plugins/aoostar-lcd.plg
   ```

   The display should light up with the default sensor panel within a few
   seconds.

3. Open **Settings → Utilities → AOOSTAR LCD** and set the network
   interface you want shown (default `br0`), then *Apply*.

4. Optional: install the **IT87 Driver** plugin (by ich777) from the Apps
   tab for fan-RPM / extra motherboard sensors.

## Customizing what the display shows

- Panel layout: `/boot/config/plugins/aoostar-lcd/cfg/monitor.json`
  (same format as official AOOSTAR-X themes, including background
  images in the same folder). Restart the driver after editing.
- Sensor mapping: `/boot/config/plugins/aoostar-lcd/cfg/sensor-mapping.cfg`
  maps panel labels to your hardware's sensor names. It is auto-generated;
  delete its first line to take manual control. Use the
  *Show available sensors* button on the settings page to list everything
  your system reports (including per-NVMe temperatures to map to the
  SSD slots on the panel).

## Why not the official AOOSTAR-X binary?

The vendor's Linux build does run on Unraid (their own readme describes a
User Scripts workaround), but it is a 182 MB closed-source PyInstaller
blob, runs a web server on port 5123 with default credentials, and has a
built-in expiration mechanism. The aoostar-rs driver is ~17 MB, open
source (MIT/Apache-2.0), and has no service/UI baggage — a much better
fit for a headless NAS. The vendor package is kept in
`../WTR MAX+GEM12+ PRO Screen Software/` as a reference/fallback.

## The "Unraid Dark" panel

The plugin ships its own panel (generated by `src/make_panel.py`) instead
of the vendor theme:

- **CPU % and GPU % side by side** with the shared die temperature below,
  colored green/amber/red by configurable thresholds (colors work via
  triple label "slots" — `unraid-stats.sh` writes each value under exactly
  one of `_g`/`_a`/`_r` and asterctl skips labels with no value)
- **Network** current up/down speeds, IP address, a clock, and per-direction
  **history sparklines** (block-element graphs auto-scaled to the window
  peak, window length configurable, default 15 min)
- **RAM** usage
- **Disk strip**: green max-temperature summary (HDD / SSD split, standby
  count); switches to a red alert listing any disk at/over its threshold.
  Temperatures come from Unraid's own `disks.ini`, so spun-down disks are
  never woken.

`unraid-stats.sh` was tested against mock data in a Linux container
(threshold colors, alert/summary switching, standby/flash handling, and
sparkline generation against ramping traffic). Note: asterctl's sensor
map never drops keys, so the collector rewrites every label each cycle
(empty = render nothing) - relying on absent keys causes stale overlaps.
Text positions may need on-device tweaking — edit
`/boot/config/plugins/aoostar-lcd/cfg/monitor.json` and restart the
driver (once customized, the plugin never overwrites it; delete it to
return to stock).
